Change Logs

v2.1.1

  • add two "snips" KWS nodes and related examples
  • refactor "PulseCollectorNode" and "AlsaCollectorNode": now the capture sample rate is fixed to 48K and the output sample rate is fixed to 16K
  • rewrite "VepAecBeamformingNode": support all MicType, support output single-beam or multi-beam and improve performance
  • rename all the KWS nodes and all the examples
  • update docs
  • fix a bug in "HybridNode"

v2.0.0

  • add multi-hotword detect and vad detect in all the KWS node
  • modify HotwordDetectionNode::HotwordDetected() interface
  • change DirectionReporterNode to DirectionManagerNode, because this node can not only report direction, but also can set direction

v1.4.0

  • add Alsa collector node
  • add 'output interleave' in BaseNode::StoreBlock, all the nodes support output interleaved or deinterleaved data now
  • add components/block_length_governor: helper to change len of deinterleaved data and use it in Hybrid node, now hybrid node is able to accept any length of data
  • add manual doa node: Design for user who need to get the speaker direction with other method(e.g. with camera) and set that direction to this node to pick the beam in that direction.
